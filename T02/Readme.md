🧩 T02: Control de versions
🛠️ Treballant amb git
📝 Breu descripció

Fins ara hem gestionat el control de versions utilitzant directament el repositori des de la web de GitHub. Tot i que ens ha ajudat força, aquesta manera de treballar té algunes limitacions importants:

⚠️ Limitacions de treballar només amb GitHub Web

🐌 Lentitud a l’hora d’editar
L’editor web no és tan potent ni còmode com un editor local de codi com Visual Studio Code o un editor de Markdown com Ghostwriter.

😵‍💫 Gestió complicada del repositori
Afegir arxius i carpetes pot resultar feixuc i requereix accions poc eficients.

🚀 Nova manera de treballar: com al món real

Per millorar tot això, començarem a treballar com es fa professionalment:

👉 Combinant:

💻 Git en local (al nostre ordinador)

☁️ Repositoris remots, en aquest cas GitHub

🔎 GitHub no és l’única opció!
També existeixen alternatives com GitLab o Bitbucket.

🧠 Una mica d’història

🧑‍💻 Git va aparèixer abans que GitHub

🔓 És un sistema de control de versions descentralitzat

🔥 Va revolucionar els sistemes centralitzats que s’utilitzaven abans

👨‍🔬 Va ser creat per Linus Torvalds, el creador de Linux

🔄 Com hi treballarem?

El nostre flux de treball serà el següent:

☁️ Partim d’un repositori a GitHub

💾 En fem una còpia local al nostre ordinador

✏️ Tots els canvis es fan en local

🔁 Seguim el flux habitual de git:

git add

git commit

⬆️ Quan acabem de treballar, pugem els canvis amb:

git push

⬇️ Quan reprenem la feina (o canviem d’ordinador):

git pull per assegurar-nos de tenir-ho tot actualitzat

📌 Important!
Això és clau si treballem des de diferents ordinadors (escola 🏫 / casa 🏠).
